# 孕期彩超系统 2.0 迁移说明

## 重要更新

系统已彻底移除对 .env 文件的依赖，所有API密钥配置改为在Web界面完成。

## 主要变更

### 1. 后端代码变更

#### 移除的代码
- `from dotenv import load_dotenv`
- `load_dotenv()`
- 所有 `os.getenv('OPENAI_API_KEY')`
- 所有 `os.getenv('STABILITY_API_KEY')`
- 所有 `os.getenv('GOOGLE_API_KEY')`
- 所有 `os.getenv('GOOGLE_CSE_ID')`

#### 修改的代码

**装饰器 `check_api_key`**:
```python
# 旧代码
if not os.getenv('OPENAI_API_KEY'):
    return jsonify({'error': 'OpenAI API密钥未配置'}), 400

# 新代码
openai_key = request.headers.get('X-OpenAI-Key')
if not openai_key:
    return jsonify({'error': 'OpenAI API密钥未配置。请在右上角点击"API配置"设置密钥。'}), 400
```

**AI生成函数**:
```python
# 旧代码
client = openai.OpenAI(api_key=os.getenv('OPENAI_API_KEY'))

# 新代码
api_key = request.headers.get('X-OpenAI-Key')
client = openai.OpenAI(api_key=api_key)
```

**Stability AI函数**:
```python
# 旧代码
stability_api_key = os.getenv('STABILITY_API_KEY')

# 新代码
stability_api_key = request.headers.get('X-Stability-Key')
```

### 2. 前端代码变更

前端代码已经支持从 localStorage 获取API密钥并传递给后端：

```javascript
// 获取API密钥
const openaiKey = localStorage.getItem('openai_api_key');

// 发送请求时在Header中传递
$.ajax({
    url: '/api/generate-ultrasound-image',
    method: 'POST',
    headers: {
        'X-OpenAI-Key': openaiKey
    },
    // ...
});
```

### 3. API路由变更

**保存API密钥**:
```python
# 旧代码：保存到 .env 文件
with open('.env', 'w') as f:
    f.writelines(env_content)

# 新代码：仅验证格式
if openai_key and not openai_key.startswith('sk-'):
    return jsonify({'error': 'OpenAI API密钥格式不正确'}), 400
```

**检查API密钥状态**:
```python
# 旧代码：检查环境变量
openai_configured = bool(os.getenv('OPENAI_API_KEY'))

# 新代码：返回前端检查提示
return jsonify({
    'openai_configured': False,  # 由前端实际检查localStorage
    'message': '请在前端配置API密钥'
})
```

## 安全性说明

1. **密钥存储**: 密钥保存在浏览器 localStorage，不会上传到服务器
2. **传输安全**: 通过HTTPS传输时，密钥在Header中加密传输
3. **隐私保护**: 服务器端不存储任何API密钥
4. **清除机制**: 清除浏览器数据会删除密钥，建议定期更换密钥

## 故障排除

### 问题1: 提示 "OpenAI API密钥未配置"
**解决**: 点击右上角 "API配置"，输入有效的OpenAI API密钥

### 问题2: 生成图像失败
**解决**:
1. 检查密钥是否正确（以 sk- 开头）
2. 检查账户余额是否充足
3. 查看浏览器控制台是否有JavaScript错误
4. 查看后端控制台是否有Python错误

### 问题3: Stability AI无法使用
**解决**: 在 "API配置" 中输入 Stability AI API密钥
